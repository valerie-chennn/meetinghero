## 一、产品现状诊断

### 核心流程盘点

| 环节 | 现状 | 评估 |
|------|------|------|
| Onboarding | 4步收集名字/等级/职位/行业 | 功能完整，流程清晰 |
| 会议来源选择 | AI生成 / 上传txt或pdf | PDF解析仅取文件名，实质未解析内容 |
| 会议生成 | 基于职位/行业/等级生成完整对话 | 核心能力，质量依赖prompt |
| 会前Briefing | 角色介绍+会议背景抽屉式展开 | 信息完整，体验良好 |
| 模拟会议 | 逐条播放+关键节点暂停+narrator机制 | 核心亮点，体验流畅 |
| 会后复盘（Review） | 称号+角色私信动效 | 情感化设计完成度高 |
| 逐节点复盘（ReviewNodes） | A1/A2/B1/B2 分级练习 | 分级差异明显，教学设计扎实 |
| 完成页 | 难度反馈+再来一场 | 过于简单，留存入口不足 |

### 关键问题识别

**体验层**
- 完成页功能薄弱——只收集难度反馈后就让用户自由了，缺少成就感沉淀和下次行动召唤
- 难度反馈数据没有被用于下一场的难度自适应，收集了等于没用
- Onboarding 每次重新开始都要重新填，老用户体验差（首页有"修改我的信息"但链接回Onboarding）
- 会议中没有进度指示——用户不知道还有几个关键节点，焦虑感会放大
- ReviewNodes 的"跳过练习"按钮暗示这个练习不够吸引人，是一个产品信号

**功能层**
- PDF 上传是假功能——只读取了文件名和大小，内容完全未解析，用户使用后会失望
- 没有历史记录——用户之前练过什么、学过哪些句式，全部清零
- 没有语音输入——输入框旁边有麦克风图标（首页展示），但实际不可用（VoiceRecorder组件存在但未集成到Meeting页面）
- narrator 文案是前端写死的几条随机句，内容单薄，长期使用后明显重复
- 角色私信只有一个角色发言，多角色反馈缺失
- 完成页"再来一场"直接跳到SourceSelect，没有基于上次表现的个性化推荐

## 二、当前版本优化建议

### 2.1 用户体验层

#### UX-01：会议进度条
**问题**：用户不知道关键节点总数，不知道什么时候会议结束，心理负担大
**方案**：在Meeting页面顶部导航栏右侧加入节点进度（如 ○●○，已完成/当前/待完成）
**优先级**：P0
**实现复杂度**：低（meetingData中已有keyNodes数量，completedNodes已在state中追踪）

#### UX-02：完成页内容强化
**问题**：完成页太空洞，用户完成一场会议后缺少成就感沉淀
**方案**：
- 显示本次会议学到的句式卡片（从reviewData.nodes提取）
- 显示累计练习场次（需要localStorage计数）
- 加入"分享战报"入口（截图分享，即使是假功能先占位也有引导作用）

**优先级**：P1
**实现复杂度**：中

#### UX-03：难度反馈实际生效
**问题**：Complete页收集了难度反馈但没有任何后续动作
**方案**：将难度反馈存入localStorage，下次生成会议时作为hint传给后端，影响会议难度（NPC发言速度/关键节点压力程度）
**优先级**：P1
**实现复杂度**：中（前端→后端→prompt改造）

#### UX-04：PDF 上传真实解析
**问题**：PDF上传是假功能，严重损害用户信任
**方案**：使用 `pdf-parse` 或 `pdfjs-dist` 在服务端提取文本内容，传给AI生成会议
**优先级**：P0
**实现复杂度**：低（服务端增加依赖，SourceSelect.jsx已有上传逻辑）

#### UX-05：语音输入集成
**问题**：VoiceRecorder.jsx 已存在但未挂载到 Meeting.jsx 的输入区
**方案**：在UserInput中集成麦克风按钮，触发Azure Speech实时识别，识别结果填入输入框
**优先级**：P1
**实现复杂度**：中（代码已有基础，需要串联Azure Speech STT）

#### UX-06：narrator 文案 AI 化
**问题**：前端写死5条随机narrator，重复使用后无新鲜感
**方案**：在生成会议时由AI生成narrator文案并写入dialogue，前端的随机注入逻辑作为兜底
**优先级**：P2
**实现复杂度**：低（在generate-meeting prompt中增加narrator生成指令）

### 2.2 功能层

#### FN-01：历史记录页
**问题**：每次会议后数据随页面刷新丢失，无法回顾
**方案**：
- 后端sessions/meetings/reviews已有数据库记录，只需前端加一个历史页
- 按时间倒序展示练习记录，点击可回看复盘报告
- 展示每次学到的句型积累

**优先级**：P0（数据已有，只差展示页面）
**实现复杂度**：中

#### FN-02：句型词库（个人积累）
**问题**：每次学到的句型没有持久化展示入口
**方案**：
- 在历史记录或个人页，展示所有练习中"掌握"的句型列表
- 支持简单的复习（翻卡确认）

**优先级**：P1
**实现复杂度**：中（reviews表已有nodes数据，解析masteredPatterns即可）

#### FN-03：多角色私信
**问题**：会后只有一个角色私信，信息维度单薄
**方案**：生成2-3个不同角色的私信（不同视角），用折叠卡片形式展示，第一条自动展开
**优先级**：P2
**实现复杂度**：低（修改generate-review prompt即可）

## 三、短期功能扩展（1-2个月）

### 3.1 留存增长类

#### NEW-01：每日一练（Daily Challenge）
**背景**：目前用户每次使用都需要完整走一遍会议流程（3-5分钟），缺少轻量级高频入口
**方案**：
- 每天推送一个"今日场景卡"，包含一个会议片段（1-2条NPC对话）和一个关键节点
- 用户只需回答一次，获得即时反馈
- 无需生成完整会议，直接从题库（或AI实时生成）取一道题
- 完成后有连续打卡计数

**用户价值**：5分钟内可完成，适合通勤时练习，大幅降低使用门槛
**商业价值**：提升DAU，建立打卡习惯
**实现复杂度**：中
**优先级**：P0（高ROI）

#### NEW-02：会议类型扩展
**背景**：目前只有"Weekly Sync（项目周会）"一种类型
**方案**：增加以下高频会议类型：
1. **1-on-1（与上级单独同步）** —— 汇报进度、寻求资源
2. **Retrospective（复盘会）** —— 说问题、提建议
3. **Stakeholder Update（给跨部门汇报）** —— 简洁表达数据
4. **Job Interview（英文面试）** —— 衍生场景，独立价值高

每种类型有对应的NPC角色预设和节点类型
**实现复杂度**：中（主要是prompt工程，前端增加SourceSelect选项）
**优先级**：P1

#### NEW-03：表达收藏夹
**背景**：用户在复盘中看到好句子，但没有收藏入口
**方案**：
- 在ReviewNodes的"更地道的说法"卡片右上角加收藏按钮
- 收藏后进入个人词库，支持随时查看和复习
- 词库支持按句型分类

**优先级**：P1
**实现复杂度**：低（前端localStorage存储即可启动）

### 3.2 学习深度类

#### NEW-04：语音输出质量反馈
**背景**：目前只有TTS（听NPC说话），没有评测用户发音
**方案**：
- 在关键节点输入区，用户选择"语音模式"后可朗读参考句
- Azure Speech Pronunciation Assessment 对发音打分
- 显示准确度/流利度评分，突出发音差的词

**用户价值**：口语能力提升，区别于只练写的工具
**实现复杂度**：中高（Azure Pronunciation Assessment API）
**优先级**：P2

#### NEW-05：错误模式追踪
**背景**：用户每次在哪类节点表现差，目前没有横向分析
**方案**：
- 跨场次统计：用户在"解释进展"类节点vs"承受压力"类节点的表现差异
- 在首页或历史页展示"你的弱项：压力节点表现较弱，建议多练"
- 生成下一场时，自动增加弱项节点的权重

**优先级**：P2
**实现复杂度**：高

### 3.3 社交传播类

#### NEW-06：战报分享图
**背景**：完成页有"分享"潜力，但目前缺失
**方案**：
- 完成后生成一张图：称号 + 今日学到的句式 + 会议主题 + 品牌水印
- 微信/朋友圈分享（h5截图方案，无需原生能力）

**优先级**：P2
**实现复杂度**：中（html2canvas 截图）

## 四、中长期产品规划

### 4.1 产品边界扩展

#### 方向一：从"会议"到"职场英语全场景"
当前定位专注会议，但目标用户的英语痛点不仅于此。扩展路径：
- **邮件写作**：给上级/客户写英文邮件，AI评分+改写建议
- **Slack/Teams 消息**：非正式职场英语，更高频
- **Presentation**：做英文演讲，每段练一句开场/过渡/结尾

这条路线可以让产品从"会议练习"升级为"职场英语助手"，提升品牌天花板。

#### 方向二：从"个人练习"到"企业培训"
目前是个人C端产品，但企业采购的市场更大：
- **企业版**：HR批量创建账号，为全组配置"会议英语培训"
- **自定义场景**：上传公司内部会议纪要，AI生成贴合公司文化的模拟场景
- **管理看板**：HR查看团队成员练习进度和薄弱项
- **定价**：按席位收费（SaaS模式）

此方向变现效率最高，是中期最值得探索的商业化路径。

#### 方向三：从"文字"到"视频面试练习"
- 模拟视频会议环境（虚假摄像头UI）
- AI面试官提问，用户回答，AI评分表情/语速/内容
- 对标 Pramp、Interviewing.io 但专注中国英语学习者

这是产品的高阶形态，技术门槛较高，适合18个月后探索。

### 4.2 商业化路径

#### 阶段一：免费增长期（当前-3个月）
- 核心流程全免费，建立用户口碑
- 完善历史记录、句型积累等增加留存的功能
- 通过分享战报获取自然增长

#### 阶段二：Freemium 转化期（3-6个月）
| 功能 | 免费版 | Pro版 |
|------|--------|-------|
| 每月会议场次 | 5场 | 无限 |
| 会议类型 | Weekly Sync | 全类型含面试 |
| 历史记录 | 最近3次 | 全量保存 |
| 发音评测 | - | 支持 |
| 难度自适应 | - | 支持 |
| 句型词库导出 | - | PDF导出 |

定价参考：Pro版 ¥29/月 或 ¥199/年

#### 阶段三：企业版（6-12个月）
- 最低采购：10席位起，¥199/席位/年
- 提供定制场景（上传公司资料）
- 提供管理看板

### 4.3 竞品差异化

| 维度 | MeetingHero | 流利说 | 英语流利星 | Elsa Speak |
|------|-------------|--------|------------|------------|
| 核心定位 | 职场会议场景练习 | 碎片化跟读/口语 | 系统课程 | 发音纠正 |
| 场景真实感 | 高（动态AI会议） | 低（标准化题目） | 低（课程体系） | 无场景 |
| 中文用户体验 | 针对设计（中英混输） | 一般 | 一般 | 差 |
| i人友好 | 核心差异（narrator机制） | 无 | 无 | 无 |
| 会后复盘深度 | 高（分级练习） | 低 | 中 | 低 |
| 内容个性化 | 高（职位/行业定制） | 低 | 低 | 无 |

**核心差异化护城河**：
1. **情境真实感** —— AI动态生成的会议比所有竞品的标准化内容更有代入感
2. **i人内心独白** —— 独特的情感设计，让内向用户感到被理解，形成情感依赖
3. **中英混输** —— 唯一允许用中文"表意"再转英文的产品，极大降低焦虑感
4. **职场场景深度** —— 不是通用英语，是"你的职位+你的行业"的精准场景

这些差异在6-12个月内难以被复制，应作为核心投入方向持续深化。

## 执行建议

### 第一个迭代冲刺（1-2周）
优先完成P0全部4项：PDF解析修复、会议进度条、历史记录页、每日一练雏形。这4项可让产品从"好玩的MVP"变成"有留存价值的产品"。

### 第二个迭代冲刺（3-4周）
推进P1功能，尤其是语音输入（UX-05）和会议类型扩展（NEW-02）。语音输入能让产品真正进入"口语练习"赛道，打开新的用户心智。

### 商业化时机
建议在MAU稳定过500后开始测试Freemium付费墙，过早付费化会阻碍口碑传播。
