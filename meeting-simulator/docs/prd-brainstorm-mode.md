# PRD：脑洞模式（Brainstorm Mode）

**版本**：v1.0  
**状态**：待开发  
**日期**：2026-04-01  

---

## 1. 产品概述

### 一句话定位

用你喜欢的角色开英文会议，好玩着就把英语练了。

### 目标用户

已有英语学习意愿、但对"职场会议"场景不感兴趣或疲倦的用户，以及想用娱乐化方式持续保持练习动力的用户。

### 核心价值

- **降低练习门槛**：用熟悉的 IP 角色消除陌生感，降低开口焦虑
- **提升复练率**：随机性和角色碰撞的惊喜感驱动用户反复尝试
- **英语能力不打折**：娱乐化包装，但对话流、复盘、分级表达等英语训练模块完全保留

## 3. 页面清单

### 3.1 首页（Home）—— 改造

**现状**：单个"开始第一场"CTA，老用户显示"再开一场"。  

**改造内容**：
- 原有单 CTA 替换为两个并列入口卡片：
  - 左/上卡片：**正经开会**——副标题"职场真实场景"，点击走原有流程
  - 右/下卡片：**脑洞模式**——副标题"IP 角色跨界开会"，点击进入 BrainstormEntry
- 两个卡片设计对等，视觉权重相同
- 首页需要 UI 设计师全新重构，不保留旧版布局。核心是两个并列入口卡片：正经开会 / 脑洞模式。
- 老用户首页：两个入口都显示，历史记录和修改信息链接保留在底部
- 新用户走 Onboarding 后同样进入两入口首页

**不变**：品牌区（MeetingHero + slogan）、历史记录入口、修改信息入口

---

### 3.2 Onboarding（花名 + 英语等级）—— 改造

**现状**：4 步（花名 → 英语等级 → 职位 → 行业）。

**改造内容**：
- **步骤数改为 2 步**：花名（step 0） → 英语等级（step 1）
- 完成后直接跳转首页，不再问职位和行业
- 职位和行业改为"选了正经开会后按需补充"（见 3.3）
- 进度条改为 2 段

**英语等级描述更新**（通用化，去掉职场相关措辞）：

| 等级 | 新描述 |
|------|--------|
| A1 | 会一些单词和短句，还不太能组织完整的表达 |
| A2 | 能说完整的句子，但词汇量有限，复杂的说不出来 |
| B1 | 日常交流没问题，想学更地道的表达方式 |
| B2 | 表达流利，想挑战更复杂的场景和高级表达 |

**createSession 调用时机变化**：  
- Onboarding 完成时只传 `{ userName, englishLevel }`，不传 jobTitle/industry  
- jobTitle/industry 在"正经开会补充信息"步骤完成后再传（或在生成会议前合并传入）

### 3.3 正经开会补充信息页（WorkInfoStep）—— 新增

- 已有值 → 直接跳 SourceSelect
- 无值 → 进入 WorkInfoStep，填写职位 + 行业（2 步，复用现有 chip 设计）

完成后保存到 AppContext 并跳转 SourceSelect。

### 3.5 角色搜索页（CharacterSearch）—— 新增（点将局专用）

**路由**：`/brainstorm/search`

**内容**：
- 标题："想跟谁开会？"
- 输入框：placeholder "输入角色、作品、或任何你想到的名字..."
- 输入框下方热门推荐标签：西游记、哈利波特、三国、漫威、甄嬛传、海贼王等

### 3.6 角色选择页（CharacterSelect）—— 新增（点将局专用）

**路由**：`/brainstorm/characters`

**内容**：
- 标题："选 2-3 位角色"
- 展示搜索结果的全部角色，每张角色卡包含：
  - 角色名称
  - 一句话人设
- 多选，2-3 个，超出 3 个禁止继续选，不足 2 个"下一步"按钮不激活
- 已选数量实时提示（如"已选 2 / 最多 3"）
- 底部"下一步"按钮

---

### 3.7 随机抽签页（RandomDraw）—— 新增（乱炖局专用）

**路由**：`/brainstorm/random`

**内容**：
- 标题："乱炖局"
- 3 张翻牌卡（初始显示背面）
- 点击"翻牌"按钮 → 依次或同时翻开，揭晓 3 个来自不同世界的随机角色
- 每张翻开的角色卡显示：角色名称、来源世界、一句话人设
- 底部两个按钮：
  - **换一批**：重新随机，重置翻牌动画（无次数限制）
  - **就这三位**：确认，进入 ThemePreview

**随机规则**：3 个角色必须来自 3 个不同世界（后端随机或前端按世界池随机）。

### 3.8 主题预览页（ThemePreview）—— 新增（点将局 + 乱炖局共用）

**路由**：`/brainstorm/theme`

**内容**：
- 展示 AI 生成的会议主题，包含：
  - 主题标题（中文）
  - 会议场景设定（2-3 句，说明在哪开、谁有实权）
  - 用户身份标签（如"天庭派来的监察仙官"）
  - 参会角色列表（含适配后的身份头衔，乱炖局显示）
- 底部按钮：
  - **换一个主题**（按钮旁显示 `x/3`，如 `1/3`、`2/3`、`3/3`，初始显示 `0/3`）
  - **进入会议**（激活状态）
- 换完 3 次后（显示 `3/3`）"换一个主题"按钮置灰

**换主题逻辑**：每次点击重新调用 AI 生成，消耗一次次数，次数存在本地 state，刷新重置。

## 5. Prompt 改造要点

### 5.1 新增函数：`generateBrainstormMeetingPrompt`

在 `server/prompts/generate-meeting.js` 中新增独立函数，与现有 `generateMeetingPrompt` 并行，不修改原函数。

**函数签名**：

```javascript
function generateBrainstormMeetingPrompt({
  englishLevel,
  userName,
  sceneType,           // 'brainstorm-pick' | 'brainstorm-random'
  characters,          // 角色对象数组，含 name/world/worldLabel/persona/tags
  mainWorld,           // 乱炖局：随机选中的主场景世界（三者之一），点将局为唯一世界
})
```

### 5.2 sceneType 分支逻辑

**点将局（brainstorm-pick）**：
- 所有角色来自同一世界，主场景 = 该世界
- 用户身份 = 该世界观中一个有实权、能拍板的角色（AI 生成）
- 会议地点和设定贴合该世界（如西游记 → 天庭南天门、贾府 → 荣庆堂）

**乱炖局（brainstorm-random）**：
- 主场景世界 = 从三个角色的世界中随机选一个（前端传入 `mainWorld`）
- 其他角色适配进主场景：保留性格和说话风格，**只改身份头衔**
  - 例：贾府场景 + 乔布斯 → 头衔改为"贾府西洋机关师"，说话风格仍是乔布斯
- 用户身份 = 主场景世界中一个有实权角色（AI 生成）

### 5.3 关键约束（写入 systemPrompt）

**必须遵守**：
1. 禁止使用现代企业设定（XX公司、XX总部、CTO、Sprint、周会等词汇）
2. 用户身份必须有实权、能参与决策，禁止"临时主持人"、"旁观者"、"记录员"等被动角色
3. 乱炖局中非主场景角色只改头衔，性格标签和说话风格保持原角色（AI 生成对话时参考 tags）
4. 会议主题必须贴合角色世界观（如西游世界 → 可讨论"东海龙宫借宝/九九八十一难的分工调度"，不能讨论"Q3 OKR"）
5. narrator 不生成（脑洞模式 systemPrompt 不含 narrator 相关指令）

### 5.4 复盘角色私信约束（写入 review prompt）

- 每个角色的私信文本必须包含该角色的语言风格标注
- 例：孙悟空的私信用"俺老孙"、"大师兄"、活泼直白的口吻；林黛玉的私信用古典诗意、多愁善感的语气
- `tags` 字段直接传入 review prompt 作为风格约束

### 5.5 用户身份生成规则（systemPrompt 示例片段）

```
用户（花名：{userName}）在本场会议中的身份：
- 必须是{主场景世界}中一个有实权、能拍板、参与核心决策的角色
- 点将局示例：西游记场景 → "天庭派来的监察仙官"
- 乱炖局示例：贾府场景 → "新任管家"
- 禁止使用：临时主持人、旁观者、记录员、见习XXX
- 用户不扮演任何已有 IP 角色
```

- [ ] 完成后跳转首页，state 中有 userName 和 englishLevel

- [ ] 填写完成后 state.jobTitle 和 state.industry 有值

### 6.4 BrainstormEntry

- [ ] 显示点将局、乱炖局两个入口

- [ ] 显示热门推荐标签，点击可快速填入搜索词

- [ ] 展示 AI 生成的会议主题标题、场景设定（2-3 句）、用户身份标签

- [ ] 换完 3 次后按钮置灰，不再可点
- [ ] 点击"进入会议"跳转 Loading → Meeting

### 6.9 Meeting（脑洞模式）

- [ ] 对话流中不出现 narrator 气泡
- [ ] NPC 说话风格贴合其原角色人设（如孙悟空用口语化表达）
- [ ] NPC 对话中无现代企业词汇（Sprint/OKR/CTO 等）
- [ ] 关键节点任务卡正常出现，结构与正经开会一致
- [ ] 用户身份有实权（检查 briefing 或角色卡描述）

### 6.10 Review（脑洞模式）

- [ ] betterWay / pattern / practice 模块正常显示
- [ ] 角色私信语气符合原角色风格（孙悟空口吻 ≠ 林黛玉口吻）
- [ ] 会议记录写入历史，scene_type 字段值为 'brainstorm-pick' 或 'brainstorm-random'

## 7. 不做的事（MVP 排除项）

1. **角色混搭**：点将局只能从一个世界选角色，不支持跨世界混选（想混搭只能用乱炖局）
2. **情节选择**：点将局 MVP 不提供情节树，AI 自动生成，用户只能换主题，不能定制情节走向
3. **角色扩充功能**：不做用户投票/申请添加角色的入口，角色池 MVP 阶段手动维护
4. **收藏角色组合**：不做收藏/分享功能
5. **付费限制**：换主题次数限制 MVP 只做基础计数，不接支付流程（付费点预留但不实现）
6. **乱炖局次数限制**：换一批无次数限制（付费点留待后续）
7. **角色语音 TTS 风格化**：所有角色共用同一 TTS 音色，不做角色差异化语音
8. **多语言支持**：角色界面只有中文，会议对话仍为英文
9. **角色详情页**：不做单独的角色介绍详情页
10. **A/B 测试框架**：不做正经开会 vs 脑洞模式的用户分流统计
