# MeetingHero 复盘教学流程重设计 PRD

**版本**: v1.0
**日期**: 2026-03-30
**作者**: 产品经理
**状态**: 待架构师 / 设计师评审

---

## 背景与问题诊断

### 现状

用户完成英文会议模拟后，进入复盘学习页，当前设计为每个关键节点走 4 步：

1. 你刚才说（展示用户原话）
2. 这里可以这样说（展示更好的表达）
3. 下次可以这么说（句型提炼）
4. 试着说一次（新场景练习）

共 3 个节点，顺序独立。

### 核心问题诊断

用户反馈"教学感很弱"，根因分析如下：

| 问题维度 | 当前状态 | 根因 |
|---------|---------|------|
| 认知负荷 | 4 步全部展示在一屏 | 用户直接滑动到底，没有主动加工 |
| 学习模式 | 纯展示（输入型） | 缺乏输出型环节，没有检索练习 |
| 步骤逻辑 | 1→2→3→4 线性堆叠 | 步骤 2 和 3 高度重叠，衔接断层 |
| 节点关系 | 3 个节点完全独立 | 缺乏整体感，不知道"学完了什么" |
| 进度感 | 无 | 用户不知道自己处于何处 |

### 教学感弱的本质

"看"和"学"的区别在于：**是否发生了主动的信息检索和输出**。

- 看：信息进入眼睛 → 感觉理解了 → 实际没有编码
- 学：被迫回忆 + 尝试输出 → 遭遇障碍 → 障碍被克服 → 真正记住

核心设计原则：**制造适当的"认知摩擦"**，让用户不得不主动参与。

---

## 约束条件

- 目标用户：A1-A2 英语水平（词汇量有限，不能有复杂判断题）
- 每个节点复盘时长：2 分钟以内
- 平台：移动端优先，375px 宽度
- 交互限制：不能有拖拽、连线等复杂操作
- 节点数量：3 个

---

## 三个方案

---

## 方案 A："先猜后看"——预测-揭示-输出模型

### 设计哲学

**猜测行为本身就是学习**。在看到正确答案之前，先让用户猜，即便猜错，大脑也会更深刻地编码正确答案（测试效应 / 生成效应）。

### 步骤设计（每节点 3 步，约 90 秒）

```
步骤 1：情境还原 + 预测（25 秒）
步骤 2：揭示 + 对比（30 秒）
步骤 3：即时复现（35 秒）
```

**步骤 1 — 情境还原 + 预测**

- 展示对话气泡：[对方说的话] + [你当时的回答]
- 底部出现问题："如果重来，你会怎么说？"
- 用户交互：从 3 个选项中选 1 个（A/B/C 选择题，选项均为英文短句）
- A1-A2 适配：选项使用简单词汇，不超过 10 个词
- 不展示对错，点选即进入步骤 2

**步骤 2 — 揭示 + 对比**

- 上半屏：你选的 vs 推荐表达，并排展示，高亮差异词
- 下半屏：1 句话解释为什么推荐表达更好（中文解释，英文示例）
- 句型标注：用标签标出核心句式，如 `[表示不确定]` `[礼貌拒绝]`
- 用户交互：点击"听一听"TTS 朗读推荐表达（必须触发才能继续，作为强制停留机制）

**步骤 3 — 即时复现**

- 展示新场景（1 句话，场景与节点相关但情境不同）
- 用户交互：点击麦克风，跟读或仿说推荐句式
- 完成后展示"你说的 vs 目标句"文字对比
- 底部：进度条 + "下一个节点"按钮

### 节点间衔接

每完成一个节点，底部弹出一张"句型卡片"（类似 Duolingo 的单词卡）：

```
你掌握了：[句型名称]
"I'm not sure about the timeline, but I can check."
```

3 个节点完成后，3 张卡片叠加显示 → "本次复盘总结"，形成闭环。

### 解锁机制

**逐步解锁，但每步等待感极短**。步骤之间无需额外点击"下一步"——完成交互动作（选择/点TTS/跟读）即自动进入下一步，过渡动画 0.3 秒。

### 为什么这样能增强教学感

1. 选择题迫使大脑提取已有知识，触发"生成效应"
2. TTS 强制点击打断被动浏览惯性
3. 句型卡片在节点末尾制造成就感，让用户感知到"学到了东西"
4. 3 步比 4 步节奏更紧凑，2 分钟内可完整走完

## 方案 B："填空驱动"——空缺-补全-输出模型

### 设计哲学

**不要展示完整的正确答案，让用户自己补全它**。填空比选择题认知负荷更高，但比自由输入更可控——适合 A1-A2 用户。

### 步骤设计（每节点 4 步，约 110 秒）

```
步骤 1：你当时说了什么（情境）（15 秒）
步骤 2：填空——补全更好的表达（35 秒）
步骤 3：完整句展示 + 读出来（25 秒）
步骤 4：换场景用一次（35 秒）
```

**步骤 1 — 你当时说了什么**

- 对话回放气泡，展示用户原话
- 底部有一行提示："有个更地道的说法，试着猜猜？"
- 无需交互，3 秒后自动进入步骤 2（或用户点击跳过）

**步骤 2 — 填空补全**

- 展示推荐句式，关键词被遮住（用下划线 + 空格占位）
- 遮住的词语量：1-2 个词，不超过 5 个字母
- 用户交互：从屏幕下方的词库（4-6 个词块）中点击正确的词填入
- 填对：绿色高亮 + 微震动反馈，填错：红色闪烁 + 保留空格等待重选
- A1-A2 适配：被遮词均为高频词，词库中有明显干扰项但不刁钻

**步骤 3 — 完整句展示 + 读出来**

- 展示填写完整的句子（无遮挡）
- 中文释义在句子下方小字显示
- 用户交互：点击"读出来"按钮，TTS 朗读后出现"跟读"麦克风
- 用户跟读（可跳过，但有提示"跟读效果更好"）

**步骤 4 — 换场景用一次**

- 新场景文字，空白输入框
- 用户交互：点击麦克风说出来，或点击"看提示"（提示为句式模板，非完整句）
- 跟读完成后自动进入下一节点

### 节点间衔接

节点间有一个"过桥"设计——每个节点结束时展示一句话的**场景关联提示**：

```
节点 1 讲的是"表达不确定"。
节点 2 讲的是"推迟决定"。这两个场景经常一起出现——你可以连着用。
```

这让 3 个节点形成内容上的逻辑关联，而非孤立的 3 段。

### 解锁机制

**全部逐步解锁**，用户完成当前步骤的交互动作后才显示下一步。但步骤 1 自动播放不需要点击，减少点击疲劳。

### 为什么这样能增强教学感

1. 填空是主动产出行为，比选择题认知负荷更高，记忆编码更深
2. 词块点击比键盘输入门槛低，适合 A1-A2 且不怕拼写错误
3. 过桥文案让节点产生语义关联，用户能感知到知识结构
4. 步骤 1 自动跳转避免了无意义的点击等待

## 方案 C："闯关积分"——挑战-通关-汇总模型

### 设计哲学

**将复盘重构为一场小型闯关**，而不是教学展示。3 个节点 = 3 关，每关有明确的通关条件。学习感弱的本质原因是用户不感到"被考验"——方案 C 正面解决这个问题。

### 步骤设计（每节点 2 步，约 80 秒）

```
步骤 1：快速回顾（20 秒）
步骤 2：挑战——时限口说（60 秒）
```

**步骤 1 — 快速回顾**

- 对话片段回放（用户原话 + 对方话语）
- 一个动画箭头指向用户原话，旁边出现"更好的说法"气泡，用卡片翻转动画揭示
- 卡片翻转前用户必须点击"翻开看看"，制造期待感
- 翻开后展示：推荐句 + 核心句式标签 + 中文解释（共 3 行信息，极度简洁）

**步骤 2 — 挑战**

- 进入挑战模式，界面变为更聚焦的深色遮罩
- 展示新场景描述（1-2 句中文描述情境）
- 倒计时 30 秒显示在顶部（非压迫性，而是激励性——"挑战自己！"）
- 用户交互：点击麦克风，说出英文回应
- 30 秒内说完：获得"通关"标记；未完成：获得"再试一次"选项（最多 2 次）
- 评判维度极度宽松（A1-A2 适配）：包含核心句式关键词即算通关

**通关反馈**

- 通关：屏幕短暂弹出绿色光晕 + "Great!" 文字
- 未通关：展示参考答案 + "下次加油"，不惩罚，直接进入下一关

### 节点间衔接

3 关设计为**难度递进**：

- 第 1 关：最贴近刚才模拟的场景（最容易迁移）
- 第 2 关：稍微不同的场景（需要一定灵活运用）
- 第 3 关：综合场景（可能同时用到前两个节点的句式）

这种递进设计让用户感知到"越来越难"的挑战曲线，形成明显的成就感弧线。

复盘结束后，展示"本场战绩"：

```
3 关全通 / 2 关通过 / 1 关通过
你的核心收获：
  - 推迟决定：I'll need to check with my team first.
  - 表达不确定：I'm not 100% sure, but...
  - 委婉拒绝：That might be tricky for us.
```

### 解锁机制

**步骤 1 和步骤 2 逐步解锁**，但步骤设计极简（只有 2 步），解锁等待感不强。

步骤 1 中"翻开看看"的点击是唯一强制交互，确保用户真的看到了推荐表达，然后立即进入挑战。

### 为什么这样能增强教学感

1. "闯关"框架从根本上改变用户心理模型：从"看教程"变成"接受考验"
2. 倒计时制造轻微紧张感，激活专注状态
3. 难度递进的 3 关设计给用户一个完整的情绪弧线（易→中→难）
4. 战绩汇总让用户在结束时有明确的"我学到了这些"的感知
5. 每节点只有 2 步，流程最短，单位时间学习密度最高

## 三方案对比

| 维度 | 方案 A（预测-揭示-输出） | 方案 B（填空驱动） | 方案 C（闯关积分） |
|------|------|------|------|
| 核心机制 | 选择题 + 猜测 | 词块填空 | 限时口说挑战 |
| 主动参与程度 | 中 | 中高 | 高 |
| 每节点步骤数 | 3 步 | 4 步 | 2 步 |
| 预计时长 | ~90 秒 | ~110 秒 | ~80 秒 |
| A1-A2 适配性 | 高 | 高 | 中（口说门槛稍高） |
| 开发复杂度 | 低 | 中（词块UI） | 中（倒计时+评判逻辑） |
| 教学感强度 | 中 | 中高 | 高 |
| 趣味性 | 中 | 中 | 高 |
| 节点关联感 | 强（句型卡片） | 强（过桥文案） | 中（递进难度） |

## 推荐方案

**首选方案 A**（MVP 阶段）

理由：
1. 开发复杂度最低，可快速上线验证
2. 选择题对 A1-A2 用户最友好，不会因为英语能力不足而受挫
3. 强制 TTS 点击是一个低成本、高效果的"打断被动浏览"机制
4. 句型卡片汇总设计可复用到其他功能模块

**迭代方向**

在方案 A 基础上，根据数据表现逐步融入方案 C 的"挑战感"元素（倒计时、通关反馈）作为 v2.0 方向。

方案 B 的词块填空可作为特定句型练习的专项功能，不纳入主流程。

## 验收标准

| 指标 | 目标值 |
|------|------|
| 节点完成率（走完全部 3 步） | >= 70% |
| 复盘页平均停留时长 | 3-5 分钟（3 个节点） |
| TTS 按钮点击率 | >= 80% |
| "跟读"参与率 | >= 50% |
| 复盘后问卷"感觉学到了东西"正向回答 | >= 65% |

---

## 风险与约束

1. **语音识别准确率**：A1-A2 用户发音不标准，步骤 3 的跟读评判必须宽松，建议只检测是否有输入，不做发音评分（MVP 阶段）
2. **TTS 强制点击**：如用户在静音场景（通勤、公共场所），强制 TTS 会造成摩擦，需提供"静音模式"跳过选项
3. **选择题设计质量**：方案 A 的选择题干扰项必须经过英语教师审核，防止 A1-A2 用户无法区分选项
