# 任务档案卡片 -- 视觉设计方案

> 页面：会前准备页（PreMeeting）
> 定位：页面唯一主视觉卡片，"剧本杀任务档案"调性
> 约束：移动端 430px 宽度，不使用深色靛蓝渐变

---

## 方案 A：「羊皮卷」-- 偏质感 / 高级感

### 设计概念

受老式档案袋、牛皮纸信封、蜡封信的启发。卡片模拟一张泛黄的任务档案文件，边缘有微妙的纸张纹理，顶部有一枚红色印章作为视觉锚点。整体传达"你收到了一份机密任务"的仪式感。

### 配色方案

| 用途 | 色值 | 说明 |
|------|------|------|
| 卡片背景 | `#FBF8F3` | 温暖的羊皮纸白，微黄调 |
| 卡片边框 | `#E8DFD0` | 1px，牛皮纸色调边框 |
| 标题文字 | `#2D2417` | 深棕黑，墨水感 |
| 正文文字 | `#4A3F35` | 中棕色，阅读舒适 |
| 次要文字 | `#8C7E6E` | 浅棕灰，辅助信息 |
| 分隔装饰 | `#D4C5B0` | 牛皮纸色，用于分隔元素 |
| 印章/强调色 | `#C03030` | 暗红色，火漆印章感 |
| 星级亮色 | `#D4A843` | 古金色，填充星 |
| 星级暗色 | `#D4C5B0` | 空星，与分隔线同色 |

### 排版细节

**难度星级区域（右上角）**
- 位置：卡片右上角，padding-top: 24px, padding-right: 24px
- "DIFFICULTY" 标签：10px / 600 / 字间距 1.2px / 全大写 / 颜色 `#8C7E6E`
- 星星：14px / 间距 2px / 填充星 `#D4A843` / 空星 `#D4C5B0`

**会议主题区域**
- 英文主题：20px / 700 / 颜色 `#2D2417` / 行高 1.3 / 字间距 -0.3px
- 中文翻译：13px / 400 / 颜色 `#8C7E6E` / 行高 1.5 / margin-top: 4px

**前情提要区域**
- "前情提要" 标签：11px / 700 / 全大写 / 字间距 1px / 颜色 `#8C7E6E`
- 正文：15px / 400 / 颜色 `#4A3F35` / 行高 1.7 / 段落感宽松

**角色信息三行**
- Emoji：15px / 固定宽度 24px
- 文字：14px / 400 / 颜色 `#4A3F35` / 行高 1.5
- 行内标签（目标/难点/盟友）：14px / 600 / 颜色 `#8C7E6E`
- 行间距：12px

### 卡片整体参数

- 内边距：24px
- 圆角：16px
- 阴影：`0 2px 8px rgba(45, 36, 23, 0.06), 0 8px 24px rgba(45, 36, 23, 0.08)`
- 边框：`1px solid #E8DFD0`

### 设计亮点

**1. 火漆印章装饰**
- 卡片左上角放置一个 36x36px 的圆形装饰，模拟蜡封印章
- 颜色 `#C03030`，内部用白色文字显示 "密" 或 "CLASSIFIED"
- 圆形带有微阴影：`0 2px 4px rgba(192, 48, 48, 0.25)`
- 不是真的蜡封凸起效果，而是扁平化的印章图形，带一点 inner shadow 模拟压痕

**2. 分隔线处理 -- 墨迹淡线**
- 不用直线。使用一个 SVG 或 CSS 实现的"笔触线"效果
- 具体做法：一条细线（1px），颜色 `#D4C5B0`，两端渐隐（opacity 从 0 到 1 再到 0）
- 中间放一个小菱形装饰符 `◆`（8px，同色），作为视觉锚点
- 整体效果像书籍中的章节分隔符

**3. 纸张质感**
- 卡片背景叠加一层极微弱的噪点纹理（CSS noise 或 tiny SVG pattern）
- opacity 约 0.03，肉眼几乎看不到但下意识感觉到"不是纯色"
- 或用 CSS：`background-image: url("data:image/svg+xml,...")` 叠加微噪点

**4. 左侧竖线装饰**
- 前情提要区域的左侧有一条 2px 竖线，颜色 `#D4A843`（古金色）
- 正文区域左移 padding-left: 14px，竖线贴在左边
- 效果类似 blockquote 的引用条，但用金色暗示"重要信息"

### 难度星级视觉处理

- 使用实心五角星 `★`
- 填充星：`#D4A843`（古金色），有微妙的 `text-shadow: 0 1px 2px rgba(212, 168, 67, 0.3)` 光泽
- 空星：`#D4C5B0`，与背景融合
- 整体排列紧凑，视觉权重不抢主标题

### 入场动画

- 卡片整体：fadeInUp，400ms，ease-out
- 印章：延迟 200ms，scale 从 0.6 到 1 + rotation 从 -15deg 到 0deg，带 spring 缓动，模拟"盖章"动作
- 三行角色信息：依次 stagger 出现，间隔 80ms

## 方案 B：「全息任务卡」-- 偏趣味 / 游戏感

### 设计概念

灵感来自科幻电影中的全息投影任务面板，但做了大量"去中二"的克制处理。核心手法是：白色毛玻璃卡片 + 品牌色系的几何装饰 + 扫描线微动效。传达"系统已为你生成了一份任务简报"的科技游戏感。

### 配色方案

| 用途 | 色值 | 说明 |
|------|------|------|
| 卡片背景 | `rgba(255, 255, 255, 0.85)` | 半透明白，毛玻璃基底 |
| 页面背景 | `#F0F0F8` | 浅靛灰，让毛玻璃有东西可以模糊 |
| 页面背景装饰 | `radial-gradient(circle at 30% 20%, rgba(129, 140, 248, 0.12), transparent 50%)` | 品牌色光斑，被卡片模糊后形成朦胧色晕 |
| 标题文字 | `var(--text-primary)` / `#1E1B4B` | 系统标准深色 |
| 正文文字 | `var(--text-primary)` | 沿用系统 |
| 次要文字 | `var(--text-secondary)` / `#64748B` | 沿用系统 |
| 强调色/装饰线 | `var(--color-brand)` / `#4F46E5` | 品牌靛蓝 |
| 强调色浅 | `var(--color-brand-light)` / `#EEF2FF` | 标签背景 |
| 星级亮色 | `var(--accent-orange)` / `#F97316` | 橙色，活力感 |
| 星级暗色 | `#E2E8F0` | 空星 |

### 排版细节

**难度星级区域（右上角）**
- 整体包裹在一个小胶囊 pill 中：背景 `#FFF7ED`（Orange-50），圆角 999px，padding 4px 10px
- 星星：12px / 间距 1px / 填充星 `#F97316` / 空星 `#E2E8F0`
- 无文字标签，纯星星，pill 造型自带语义

**会议主题区域**
- 英文主题：20px / 700 / 颜色 `#1E1B4B` / 行高 1.3 / 字间距 -0.3px
- 中文翻译：13px / 400 / 颜色 `#64748B` / 行高 1.5 / margin-top: 4px

**前情提要区域**
- "BRIEFING" 标签：放在一个小标签 tag 中
  - 背景 `#EEF2FF`，颜色 `#4F46E5`，10px / 600 / 全大写
  - 圆角 4px，padding 3px 8px
  - 左侧可加一个 6px 圆形脉冲动画点（品牌色），模拟"信号接收中"
- 正文：15px / 400 / 颜色 `#1E1B4B` / 行高 1.7

**角色信息三行**
- 与方案 A 参数相同，但颜色回归系统标准色
- 行内标签颜色：`#64748B`

### 卡片整体参数

- 内边距：24px
- 圆角：20px
- 毛玻璃效果：`backdrop-filter: blur(20px) saturate(1.2)`
- 边框：`1px solid rgba(255, 255, 255, 0.6)`（玻璃高光边）
- 阴影：`0 4px 24px rgba(30, 27, 75, 0.06), 0 1px 3px rgba(0, 0, 0, 0.04)`

### 设计亮点

**1. 毛玻璃质感 + 背景光斑**
- 页面背景放 2-3 个大尺寸（200-300px）模糊圆形渐变色块
  - 左上：品牌靛蓝 `rgba(129, 140, 248, 0.12)`
  - 右下：青绿 `rgba(20, 184, 166, 0.08)`
  - 中偏右：紫色 `rgba(139, 92, 246, 0.06)`
- 卡片的 backdrop-filter 模糊这些色块，形成朦胧的彩色光晕
- 效果：卡片不是"死白色"，而是随位置微微带有色彩倾向

**2. 分隔线处理 -- 扫描线**
- 分隔使用一个高度 1px 的渐变条：
  - `linear-gradient(90deg, transparent 0%, #4F46E5 20%, #818CF8 50%, #4F46E5 80%, transparent 100%)`
  - opacity: 0.25，视觉上是一条品牌色调的淡淡光线
- 可选增强：一个 2px 高的"扫描光"从左到右缓慢移动（8s 循环），opacity 0.3
  - 只在分隔线第一次出现时播放一次，不循环——避免分散注意力

**3. 角标编号**
- 卡片右上角（星级上方），用极淡的大号文字显示任务编号
  - 例如 "MISSION-07"
  - 48px / 800 / 颜色 `rgba(79, 70, 229, 0.06)`
  - 纯装饰，增加"系统生成文件"的味道
  - 位置：绝对定位，top: 12px, right: 16px

**4. 角色信息区域的左侧彩色指示条**
- 三行信息各带一个 3px 宽的左侧色条（圆角 2px）
- 目标行：`#10B981`（翡翠绿）
- 难点行：`#F59E0B`（琥珀色）
- 盟友行：`#4F46E5`（品牌靛蓝）
- 每行信息包裹在一个 padding-left: 12px 的区域中，色条贴左
- 效果：信息分类一目了然，色彩丰富但不刺眼

### 难度星级视觉处理

- 星星包裹在一个 pill 胶囊中，整体像一个 badge
- 填充星用橙色 `#F97316`，带微弱光晕 `drop-shadow(0 0 2px rgba(249, 115, 22, 0.4))`
- 空星用 `#E2E8F0`，几乎隐入背景
- pill 背景 `#FFF7ED`（极浅橙），与橙色星星色系统一

### 入场动画

- 背景光斑：在卡片出现前 200ms 就开始 fadeIn，提前铺设氛围
- 卡片：fadeInUp + scale(0.98 -> 1)，400ms，ease-out
- 分隔扫描线：卡片出现后 300ms，一道光从左扫到右（600ms）
- 角色信息三行：stagger 出现，间隔 80ms，左侧色条从高度 0 展开到 100%

## 两个方案对比

| 维度 | 方案 A「羊皮卷」 | 方案 B「全息任务卡」 |
|------|------------------|---------------------|
| 调性 | 复古 / 实体感 / 仪式感 | 未来 / 数字感 / 科技感 |
| 质感 | 纸张纹理 + 印章 + 墨线 | 毛玻璃 + 光斑 + 扫描线 |
| 色调 | 暖色系（米白/棕/金/暗红） | 冷色系（品牌靛蓝/白/灰） |
| 与设计系统的关系 | 独立色板，与系统差异较大 | 大量复用系统 Token，一致性强 |
| 沉浸感来源 | 材质隐喻（纸张、印章、墨水） | 数字隐喻（全息、扫描、信号） |
| 游戏感强度 | 中等（剧本杀 / 密室档案） | 中等（太空任务简报 / RPG） |
| 实现复杂度 | 中（SVG 分隔线 + 噪点纹理） | 中高（毛玻璃 + 光斑 + 扫描动画） |
| 潜在风险 | 暖色系偏离主品牌调性 | backdrop-filter 在低端安卓上可能降级 |

## 设计师建议

**推荐方案 B「全息任务卡」**，理由：

1. **与现有设计系统一致性更好**。方案 B 大量复用了 `--color-brand`、`--text-primary` 等 Token，不会在页面切换时产生视觉断裂。
2. **用户明确拒绝了深色靛蓝渐变**，说明对"暗色沉重"的方向反感。方案 B 的毛玻璃是轻盈明亮的，方向更安全。方案 A 虽然不是深色，但暖棕色板与整个 App 的冷靛蓝品牌调性可能冲突。
3. **毛玻璃 + 微动效在 2026 年的移动端已是成熟方案**，降级兼容也简单（去掉 blur，加不透明白底即可）。
4. **角色信息的彩色指示条**是一个低成本高回报的设计手法，让信息层次无需依赖 emoji 解读就能一眼区分。

如果团队偏好更强烈的沉浸感和游戏感，可以选 A。但从产品一致性角度，B 是更稳妥的选择。

## 需要前端配合的技术要点

1. **方案 B 的毛玻璃**：需要 `backdrop-filter` 支持。建议用 `@supports` 检测，不支持时回退到 `background: rgba(255, 255, 255, 0.95)`
2. **背景光斑**：用绝对定位的 div + radial-gradient 实现，注意 z-index 层级关系
3. **分隔扫描线动画**：CSS `@keyframes` + `background-position` 位移，建议只播放一次
4. **方案 A 的 SVG 分隔线**：需要一个 inline SVG 或 CSS 渐变模拟笔触效果
5. **印章盖章动画**（方案 A）：CSS transform + spring 缓动，无需 JS
6. **两个方案共通**：stagger 动画建议用 CSS `animation-delay` 实现，不依赖 JS 库
