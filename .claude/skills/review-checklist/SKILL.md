# 代码审查检查清单

根据当前代码变更生成针对性的审查检查清单。

## 步骤

1. 获取当前变更：

```bash
git diff --stat HEAD 2>/dev/null || echo "无 Git 变更"
```

2. 分析变更类型，生成对应的检查清单：

### 通用检查
- [ ] 代码逻辑正确，无明显 Bug
- [ ] 错误处理完善，无未捕获异常
- [ ] 无硬编码的敏感信息（密码、密钥、Token）
- [ ] 代码注释清晰且为中文
- [ ] 变量和函数命名有意义

### 安全检查
- [ ] 用户输入已验证和清理
- [ ] 无 SQL 注入风险
- [ ] 无 XSS 漏洞
- [ ] 敏感数据已加密处理
- [ ] API 接口有权限控制

### 性能检查
- [ ] 无 N+1 查询问题
- [ ] 无不必要的循环或重复计算
- [ ] 大数据量操作有分页处理
- [ ] 无内存泄漏风险

### 测试检查
- [ ] 新功能有对应的单元测试
- [ ] 边界条件已测试
- [ ] 异常流程已测试
- [ ] 现有测试未被破坏

### 架构检查
- [ ] 符合项目架构规范
- [ ] 模块职责清晰，无越界调用
- [ ] 依赖方向正确

当前变更摘要：
```
!`git diff --name-only HEAD 2>/dev/null | head -20 || echo "无变更"`
```
